# Sites & Output

## Structure

Each generated business gets a folder in `sites/` named by slugified business name:

```
sites/
├── example/              # Reference/demo site (plumbing business)
│   ├── site-data.json
│   └── index.html
├── the-good-people/
│   ├── site-data.json
│   └── index.html
└── haley-s-beauty-therapy/
    ├── site-data.json
    └── index.html
```

## Pattern

- `site-data.json` — Source data generated by Claude API via `create-site.js`
- `index.html` — Final output compiled by `generate.js` (fully self-contained, all CSS/JS inlined)

## Regeneration

To regenerate HTML from existing data:
```bash
cd engine && node generate.js ../sites/<slug>/site-data.json
```

## Preview

```bash
npx serve sites/<slug>
```

Or use the Claude Code launch config (`.claude/launch.json`) which serves `sites/example` on port 3456.
